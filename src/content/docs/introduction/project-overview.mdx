## What you will build

This walkthrough documents the [event-driven-serverless-platform-demo](https://github.com/jajera/event-driven-serverless-platform-demo) — a consolidated <Tooltip term="aws" label="AWS" /> event-driven serverless platform for <Tooltip term="gnss" label="GNSS" /> <Tooltip term="rinex" label="RINEX" /> ingest, <Tooltip term="pytecg" label="PyTECGg" /> <Tooltip term="tec" label="TEC" /> calibration, and interactive visualization.

By the end you will have:

- **Ingest** — hourly <Tooltip term="rinex" label="RINEX" /> synced from <Tooltip term="geonet" label="GeoNet" /> into a private <Tooltip term="data-lake-bucket" label="data lake" />
- **Processing** — <Tooltip term="sqs" label="SQS" />-driven <Tooltip term="processor-lambda" label="processor Lambda" /> container writing calibrated TEC output
- **Presentation** — <Tooltip term="api-gateway" label="REST API" /> and an <Tooltip term="amplify" label="Amplify" /> portal for charts, maps, and reprocess jobs
- **Operations** — <Tooltip term="cloudwatch" label="CloudWatch" /> alarms, <Tooltip term="sns" label="SNS" /> notifications, and the `event-driven-platform` dashboard

Everything lives in one monorepo and is provisioned through <Tooltip term="terraform" label="Terraform" />.

## Architecture and data flow

![Architecture diagram: GeoNet RINEX flows through EventBridge Scheduler and ingest Lambda into S3, SQS-driven processor Lambda writes TEC output, and the Amplify portal queries via API Gateway and Lambda](/aws-event-driven-serverless-walkthrough/architecture-diagram.svg)

Data moves left to right across three layers — ingest, processing, and presentation. New raw files in <Tooltip term="s3" label="S3" /> enqueue the <Tooltip term="process-queue" label="process queue" />; operator reprocess jobs use a separate <Tooltip term="reprocess-queue" label="reprocess queue" /> with lower concurrency so scheduled ingest is not starved.

The browser and CLI reach processed data only through <Tooltip term="api-gateway" label="API Gateway" /> — not S3 or <Tooltip term="dynamodb" label="DynamoDB" /> directly. [Infrastructure Layers](/aws-event-driven-serverless-walkthrough/architecture/infrastructure-layers/) covers each component; [System Overview](/aws-event-driven-serverless-walkthrough/architecture/system-overview/) covers Terraform modules and design decisions.

## What you will see

### Hosted portal

![GNSS TEC Platform on Amplify showing station browser, VTEC time series, and ionospheric pierce point map for AUCK](/aws-event-driven-serverless-walkthrough/portal-screenshot.png)

The portal browses processed <Tooltip term="station" label="stations" />, charts TEC metrics over time, and maps <Tooltip term="ipp" label="ionospheric pierce points" />. See [Portal Usage](/aws-event-driven-serverless-walkthrough/usage/portal-usage/) and [Amplify Portal](/aws-event-driven-serverless-walkthrough/deploy/amplify-portal/).

### Operations dashboard

![CloudWatch dashboard event-driven-platform showing alarm status, SQS queue depth, processor Lambda throughput, ingest sync, and presentation API metrics](/aws-event-driven-serverless-walkthrough/cloudwatch-dashboard-screenshot.png)

Observability spans all layers. The **`event-driven-platform`** dashboard aggregates alarm health, queue backlog, Lambda throughput, and API activity. Subscribe to SNS and verify alarms in [Alarms and Dashboard](/aws-event-driven-serverless-walkthrough/verification/alarms-dashboard/).
