From the [demo repository](https://github.com/jajera/event-driven-serverless-platform-demo) root, initialize <Tooltip term="terraform" label="Terraform" /> before creating any <Tooltip term="aws" label="AWS" /> resources. This walkthrough uses `-chdir=terraform` for all Terraform commands.

## Clone and initialize

```bash
git clone https://github.com/jajera/event-driven-serverless-platform-demo.git
cd event-driven-serverless-platform-demo

terraform -chdir=terraform init
```

Terraform downloads the AWS and archive providers (requires Terraform >= 1.6, AWS provider >= 5.0).

## Review the plan

After init, preview what will be created:

```bash
terraform -chdir=terraform plan -var="region=ap-southeast-2"
```

## Deployment order

1. [ECR and Processor Image](/aws-event-driven-serverless-walkthrough/deploy/ecr-processor-image/) — create ECR repo and mirror GHCR image
2. [Staged Apply](/aws-event-driven-serverless-walkthrough/deploy/staged-apply/) — apply processing, then scheduler, then full stack
3. [Amplify Portal](/aws-event-driven-serverless-walkthrough/deploy/amplify-portal/) — confirm or manually deploy the frontend
4. [CORS Lockdown](/aws-event-driven-serverless-walkthrough/deploy/cors-lockdown/) — restrict API Gateway CORS to the portal hostname

## Verify init succeeded

```bash
terraform -chdir=terraform version
terraform -chdir=terraform providers
```

Both commands should exit 0 with provider versions listed.
