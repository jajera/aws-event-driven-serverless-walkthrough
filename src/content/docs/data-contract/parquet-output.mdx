## Query API preference

The <Tooltip term="query-api" label="Query API" /> reads `.parquet` files when `pyarrow` is available in the <Tooltip term="lambda" label="Lambda" /> runtime. It falls back to `.json` when `pyarrow` is absent. Both formats contain the same 11-column schema.

## NaN handling

The processor may emit `NaN` for fields such as `lat_ipp` and `lon_ipp` in edge cases. The Query API sanitizes these to JSON `null` before returning them, ensuring spec-compliant responses for browser clients.

## Example query response row

```json
{
  "epoch": "2024-05-29T01:00:00Z",
  "sv": "G01",
  "id_arc": 1,
  "lat_ipp": -36.85,
  "lon_ipp": 174.76,
  "azi": 45.2,
  "ele": 30.1,
  "bias": 0.5,
  "stec": 12.3,
  "vtec": 8.7,
  "veq": 9.1
}
```

```bash
aws s3 ls "s3://$(terraform -chdir=terraform output -raw bucket_name)/processed/tec/" --recursive | grep parquet
```
