Manual ingest invokes the <Tooltip term="ingest-sync" label="Ingest_Sync_Lambda" /> directly, bypassing the <Tooltip term="eventbridge-scheduler" label="EventBridge Scheduler" />. Use it for immediate data population or recovery.

## Invoke ingest-sync

From the demo repo root:

```bash
aws lambda invoke \
  --function-name ingest-sync \
  --payload '{}' \
  /dev/stdout
```

## What it does

On each invocation the <Tooltip term="lambda" label="Lambda" />:

1. Reads <Tooltip term="lookback-hours" label="LOOKBACK_HOURS" /> (default 1) to compute a rolling UTC window
2. Lists objects from `s3://geonet-open-data/gnss/rinexhourly/`
3. Copies new files to `raw/rinexhourly/{year}/{doy}/{filename}` in the <Tooltip term="data-lake-bucket" label="data lake" />
4. Skips files already present

## Verify results

```bash
aws s3 ls "s3://$(terraform -chdir=terraform output -raw bucket_name)/raw/rinexhourly/" --recursive | tail -5

aws sqs get-queue-attributes \
  --queue-url "$(terraform -chdir=terraform output -raw queue_url)" \
  --attribute-names ApproximateNumberOfMessages
```

New raw files trigger <Tooltip term="s3" label="S3" /> notifications to <Tooltip term="process-queue" label="Process_Queue" />. The <Tooltip term="processor-lambda" label="processor Lambda" /> consumes messages and writes processed output.
